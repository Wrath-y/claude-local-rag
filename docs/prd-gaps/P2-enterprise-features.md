# RAG 生产化 P2 — 多用户与企业能力

> 优先级：P2（团队/企业场景必备）
> 期望交付：P0 稳定后启动，4-6 个迭代周期
> 编写日期：2026-04-30

## 背景

现阶段服务绑定 `127.0.0.1`，无鉴权、无租户隔离、无审计、无管理界面。任何多人共享部署、SaaS 化或团队内网暴露的场景都不可行。本期目标是把工具从「个人本地」推到「团队/组织级」可用。

## 目标

- 多用户可独立管理各自知识库，数据互不可见
- 管理员可查看全局使用情况、审计历史
- 具备基础的 API 保护（鉴权、限流、滥用检测）
- 提供 Web 管理面板，不再依赖 CLI 斜杠命令

## 需求列表

### 需求 1：用户鉴权与 API Key

**模块**：鉴权

**核心功能**：

- API Key 模型：每个用户可生成多个 key，支持命名、过期、禁用
- 所有 `/ingest`、`/retrieve`、`/sources` 等端点必须携带有效 key
- `/health` 和 `/metrics` 保留无鉴权（或独立 metrics key）
- Hook 脚本从本地配置读取 key，`/rag` 系列命令透明带上

**验收**：

- 未携带 key 返回 401；错误 key 返回 403
- key 撤销后立即生效（≤ 5s）

**涉及文件**：`server.py`、`.claude/hook_script.py`、新增 `auth/`

---

### 需求 2：多租户隔离

**模块**：存储/鉴权

**核心功能**：

- 以 `tenant_id` 隔离数据：chunks 元数据增加 tenant_id，检索时强制过滤
- 存储层支持按租户分目录：`data/<tenant_id>/index.bin`
- 默认租户 `default` 兼容存量数据迁移
- 跨租户共享知识库机制：显式「共享库」概念，读权限可配置

**验收**：

- 租户 A 的 /sources 不出现租户 B 的来源
- 即使直接调用 `/retrieve` 传造假 source 名，也无法读到其他租户数据

**涉及文件**：`server.py`，新增 `storage/tenant.py`

---

### 需求 3：RBAC 与来源级权限

**模块**：鉴权

**核心功能**：

- 角色：owner / editor / viewer
- 来源（source）级 ACL：指定用户/组可读可写
- 管理员角色可全局管理用户、租户、key
- 所有权限检查在服务端执行，不信任前端/hook

**验收**：

- viewer 调用 `/ingest` 返回 403
- 将 source A 的权限撤销后用户检索不再返回其 chunk

**涉及文件**：`server.py`，新增 `auth/rbac.py`

---

### 需求 4：审计日志

**模块**：合规

**核心功能**：

- 所有写操作（ingest/delete/reset/restore）与敏感读（export、跨租户访问）记录审计事件
- 字段：timestamp、user、tenant、action、target、result、request_id
- 默认保留 90 天，存独立 SQLite/JSONL，不与业务数据混放
- 提供 `/audit/query` 检索接口（管理员）

**验收**：

- 所有上述操作在审计库留痕、不可篡改（append-only）
- 按用户/时间范围查询在 1s 内返回

**涉及文件**：新增 `audit/`

---

### 需求 5：API 限流与滥用保护

**模块**：网关

**核心功能**：

- 按 key + IP 双维度限流，默认 60 req/min、`/ingest` 10 req/min
- 大文件 ingest 额外限制：单文件 ≤ 50MB、单日总量 ≤ 1GB/用户
- 超限返回 429，Header 附 Retry-After
- 可配置黑名单（IP/key）

**验收**：

- 压测超过阈值后 1s 内触发 429
- 限流数据持久化，重启后计数连续

**涉及文件**：`server.py`，新增 `ratelimit/`

---

### 需求 6：Web 管理面板

**模块**：前端

**核心功能**：

- 登录（账号密码或 SSO 占位）
- 概览：chunk 总量、检索 QPS、存储占用、最近备份时间
- 来源管理：列表、搜索、删除、重新入库、权限配置
- 用户/租户/Key 管理
- 审计日志查看、导出 CSV
- 评测报告查看（关联 P1 需求）

**验收**：

- 面板可完成原 CLI 90% 日常操作
- 核心页面首屏 ≤ 2s

**涉及文件**：新增 `web/`（React/Vue，任选一方后续评审）

---

### 需求 7：SSO 集成占位

**模块**：鉴权

**核心功能**：

- 预留 OIDC 接入点（支持飞书、Google、GitHub、企业 IdP）
- 用户首次登录自动创建租户或加入已有租户
- JWT 校验与 API Key 并存

**验收**：

- 配置飞书 OIDC 后可登录并拿到 API Key
- 未配置 SSO 时回退到账号密码（或纯 API Key）模式

**涉及文件**：`auth/oidc.py`

---

### 需求 8：数据导出与 GDPR 合规

**模块**：合规

**核心功能**：

- 用户可导出自己所有数据（chunks 原文 + 向量 + 审计记录）为 zip
- 支持「遗忘权」：删除用户及其独占租户数据，审计记录匿名化保留
- 提供 `retention.policy`：指定来源可配置 TTL 自动清理

**验收**：

- 导出包包含所有可识别数据、附 manifest
- 执行遗忘后 30s 内检索不再命中该用户数据

**涉及文件**：`server.py`、`auth/`
