# RAG 生产化 P3 — 进阶能力

> 优先级：P3（按需选做）
> 期望交付：在 P0/P1/P2 完成后按业务需求选择推进
> 编写日期：2026-04-30

## 背景

当 P0-P2 落地后，系统已具备生产可用基线、质量可控、多租户安全。本期聚焦「拉开竞争差距」的进阶能力：多模态、Agentic 检索、自动同步、知识图谱、第三方连接器。每条需求可独立立项，不强依赖彼此。

---

## 需求列表

### 需求 1：Agentic 多跳检索

**模块**：检索

**背景**：单轮检索对「A 是什么？A 的依赖 B 在哪实现？」这类链式问题命中率低。

**核心功能**：

- 引入检索 planner：基于首轮结果判断是否需要二跳查询
- 支持最多 3 跳，拒绝环路
- 所有中间步骤可解释，返回检索轨迹
- 触发条件可配置：显式 `--agentic`、或模型置信度低于阈值

**验收**：

- 对多跳基准集 recall@3 提升 ≥ 8%
- 延迟 p95 ≤ 3s（3 跳）

**涉及文件**：新增 `retrieval/agentic.py`

---

### 需求 2：多模态支持（图片、表格、图表）

**模块**：索引/检索

**核心功能**：

- PDF/Office 文档中图片 OCR + 图像 embedding，独立索引
- 表格结构化提取，按行/列切片
- 检索结果支持附带原图预览
- 向量模型：图像用 CLIP 类模型，文字用现有

**验收**：

- 截图为 query 可检索到相关图片 chunk
- 表格行级检索在基准集命中率 ≥ 70%

**涉及文件**：新增 `indexing/multimodal/`

---

### 需求 3：自动文档同步连接器

**模块**：数据管道

**核心功能**：

- 飞书：订阅文档变更事件，自动触发 rag-update
- Notion / Confluence：定时 pull，diff 后增量更新
- Git 仓库：按 branch/path 订阅，commit 触发入库
- 本地文件夹：inotify/FSEvents 监听，文件变更自动入库

**验收**：

- 文档编辑后 ≤ 60s 内知识库同步
- 连接器故障有告警，不影响其他连接器

**涉及文件**：新增 `connectors/`

---

### 需求 4：知识图谱增强

**模块**：索引

**核心功能**：

- 从文档中抽取实体与关系（配置化 schema）
- 构建图结构，检索时作为召回补充
- 支持图检索 + 向量检索融合打分

**验收**：

- 对实体/关系型查询（"X 和 Y 的关系"）命中率提升 ≥ 10%
- 图构建作为异步任务，不阻塞 ingest

**涉及文件**：新增 `indexing/kg/`

---

### 需求 5：检索结果缓存

**模块**：性能

**核心功能**：

- 查询归一化后对 top-k 结果做 LRU 缓存
- 写操作（ingest/delete）触发对应 source 的缓存失效
- 缓存命中率暴露为指标

**验收**：

- 高频重复查询 p95 延迟下降 ≥ 50%
- 数据变更后 10s 内旧缓存失效

**涉及文件**：`server.py`，新增 `retrieval/cache.py`

---

### 需求 6：反馈闭环与在线学习

**模块**：评测

**核心功能**：

- 用户可对检索结果打标：👍/👎/忽略
- 标注数据回流到 `eval/feedback.jsonl`，可用于：
  - 扩充评测集
  - 训练 rerank 模型
- 周期性重训 rerank 模型（人工触发），验证收益后切换

**验收**：

- 标注流程 ≤ 2 次点击
- 重训模型在评测集 MRR 提升 ≥ 3% 才上线

**涉及文件**：`web/`、新增 `training/`

---

### 需求 7：分布式部署与水平扩展

**模块**：架构

**核心功能**：

- 索引后端可替换：FAISS / Milvus / Qdrant
- 元数据层可替换：SQLite / Postgres
- 无状态服务节点，支持多副本负载均衡
- 配置化路由：按租户/来源 hash 到分片

**验收**：

- 3 节点部署承压 500 QPS，p95 ≤ 500ms
- 单节点故障不影响整体可用性

**涉及文件**：架构级重构，新增 `deploy/`

---

### 需求 8：成本与用量分析

**模块**：运营

**核心功能**：

- 按租户/用户统计：存储占用、检索次数、注入 token 数
- 生成月度账单 / 用量报表
- 阈值告警：接近配额时通知

**验收**：

- 管理面板可查看任意租户 30 天趋势
- 报表可导出 CSV

**涉及文件**：`web/`、`audit/`

---

## 选型建议

P3 各需求彼此独立，建议依据实际业务信号选择推进：

- 内部文档密度高 → 优先「自动同步连接器」
- 客服/咨询场景 → 优先「Agentic 多跳」+「反馈闭环」
- 产品/设计相关 → 优先「多模态」
- 高并发对外服务 → 优先「分布式部署」+「缓存」
